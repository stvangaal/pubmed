# Blog Config

## Status
draft

## Version
v0

## Description
User-facing configuration for blog publishing. Controls the site metadata, URL structure, publishing behavior, and template paths. Users edit this as a YAML file without touching source code.

## Schema

```yaml
# config/blog-config.yaml

# Site metadata (used in templates and Jekyll _config.yml)
site_title: "Stroke Literature Weekly"
site_description: "A curated weekly digest of practice-changing stroke literature from PubMed."

# Base URL for the published site (no trailing slash)
# GitHub Pages default: https://<username>.github.io/<repo>
base_url: "https://stvangaal.github.io/pubmed"

# Whether to actually push to gh-pages (set false for local dev / dry runs)
publish: true

# Branch to push to
branch: "gh-pages"

# Directory within the branch where digest pages are stored
digests_dir: "digests"

# Closing text for blog pages (may differ from email closing)
closing: >-
  This digest was generated by the PubMed Stroke Monitor pipeline.
  Feedback on individual articles is welcome — use the feedback links above.

# Template files (relative to repo root)
templates:
  post: "config/templates/blog-post.md"
  index: "config/templates/blog-index.md"
```

```python
@dataclass
class BlogTemplatesConfig:
    post: str               # Path to blog post template
    index: str              # Path to index page template

@dataclass
class BlogConfig:
    site_title: str         # Title for the blog site
    site_description: str   # One-line description
    base_url: str           # Full base URL (no trailing slash)
    publish: bool           # Whether to push to gh-pages
    branch: str             # Target branch (default: "gh-pages")
    digests_dir: str        # Directory for digest pages (default: "digests")
    closing: str            # Closing text for blog pages
    templates: BlogTemplatesConfig
```

## Constraints

- `base_url` must not end with a trailing slash
- `branch` must be a valid git branch name
- `digests_dir` must be a relative path (no leading slash)
- `templates.post` and `templates.index` must point to existing files
- `publish: false` skips git operations but still renders pages locally
